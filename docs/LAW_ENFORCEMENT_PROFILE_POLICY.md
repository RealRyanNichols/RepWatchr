# Law Enforcement Profile Policy

RepWatchr may profile public safety agencies and badge-holding public officials when the profile is limited to public-role records, official sources, policies, budgets, public statements, public filings, public meeting records, and correction-safe source trails.

## Public Boundary

Every law enforcement, badge, sheriff, police chief, constable, public information officer, and agency-official profile must show this boundary:

> This profile covers public-role records only. RepWatchr does not publish private home addresses, minor children, or harassment instructions. Submit corrections or public sources below.

## Allowed Profile Scope

- Official agency page, directory, bio, or public role source.
- Agency policy/manual source.
- Public information request page.
- Public budget, procurement, contract, or public meeting record.
- Public statement, official press release, or public records response.
- Campaign finance only if the role is elected and the record is a public filing.
- Public disciplinary record only when official, public, lawful, and admin reviewed.
- Public case or court references only when source-backed, cautious, and reviewed.

## Prohibited Public Content

- Private home addresses.
- Minor children or family details.
- Personal phone numbers, private emails, private medical details, bank details, or unrelated private personal data.
- Threats, harassment instructions, or "show up at" language.
- Wanted-poster, bounty, target-list, mugshot, red-siren, or vigilante styling.
- Unsupported criminal accusations.
- Claims that imply guilt, corruption, or misconduct beyond the attached public source.

## Required Labels

Admin-staged public-role items must use one of these labels:

- Confirmed public record
- Source-backed public record
- Public question
- Under review
- Needs source

If the item is not source-backed, it should be framed as a public question or left under review.

## Incident And Case Mentions

Incident/case mentions can appear only when:

- An official public source or lawful public filing is attached.
- The source has a confidence/status label.
- The item is admin approved.
- The copy distinguishes what the source appears to show from what it does not prove.
- The wording avoids guilt, targeting, or harassment language.

If those conditions are not met, the public module must show the empty-state message instead of the allegation.

## Correction And Safety CTAs

Each badge/public-safety profile must include:

- Request correction
- Report private information
- Submit official source
- Report broken link

User submissions do not publish automatically.

## Analytics

The module tracks:

- `badge_profile_open`
- `public_role_boundary_viewed`
- `agency_policy_source_clicked`
- `public_info_source_clicked`
- `badge_profile_correction_clicked`
- `badge_profile_source_submitted`
- `safety_report_submitted`

These events measure product usage and source engagement, not private political targeting.
